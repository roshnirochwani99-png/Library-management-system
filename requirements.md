Functional Requirements

FR1 – Add Book: System shall allow librarian to add a new book with title, author, category, ISBN, total copies.

FR2 – View/Search Books: System shall allow librarian to view all books and search by title, author, or category.

FR3 – Manage Members: System shall allow librarian to add and view library members with basic details (name, email, phone).

FR4 – Issue Book: System shall allow librarian to issue a book to a member with issue date and due date.

FR5 – Return Book: System shall allow librarian to record return of an issued book and update availability.

FR6 – View Issued Books: System shall allow librarian to see a list of all currently issued books with member names and due dates.


Non-Functional Requirements

NFR1 – Usability: UI should be simple and understandable for non-technical staff.

NFR2 – Performance: Searching books or members should complete within 2 seconds for up to 5000 records.

NFR3 – Reliability: All data must be stored in a persistent database so it is not lost when the system restarts.

NFR4 – Maintainability: Code should be modular with clear separation of frontend, backend, and database layers.

NFR5 – Security: Only authorized users (librarian) can access the system (if you later add login).
